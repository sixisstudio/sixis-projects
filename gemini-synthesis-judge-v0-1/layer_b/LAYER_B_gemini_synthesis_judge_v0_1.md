## FORCED RULES (project-specific)

Inherits all dashboard_v0_1 + project_onboarding_v0_1 forced rules plus Universal Shell K1–K7, M-Imperatives, and ratified amendments. Project-specific rules:

**FORCED_RULE_GJ_01 — Judge is dual-mode: K3-AUDITOR + K2-RELIEF-ADJUDICATOR.** *(Amended 2026-05-08 by Tier-3 ratification of FORCED_RULE_GJ_09.)*
- **Audit mode (original scope):** The Judge observes the OUTPUT of council deliberation and may flag contradictions, unaddressed counter-arguments, or premature consensus. In audit mode the Judge does not vote, does not break ties, does not produce convergence, and does not appear in cross-poll relay rounds. Brains MUST NOT cite Judge findings during their deliberation — Judge fires after convergence, before Tommy ratifies.
- **Adjudicator mode (added per FORCED_RULE_GJ_09):** When the orchestrator faces a Tier-1 or Tier-2 decision that would otherwise be surfaced to Tommy as an option-selection ("A or B?"), the orchestrator instead routes the decision to the Judge with explicit criteria. The Judge either (a) decides directly and returns a verdict, or (b) declines and fires a council cross-poll whose convergence returns to the Judge for ratification. In adjudicator mode the Judge IS a decision-maker — but only on Tier-1/2 work, and only when explicitly routed by the orchestrator.

*Why:* Tommy's K2 sovereign authority is the scarce resource — it must be reserved for Tier-3 irreversible kernel-level decisions, not consumed on routine option-selection or cadence questions. The Judge + council are the appropriate adjudicators for Tier-1/2 work. *How to apply:* on every non-Tier-3 decision, route to Judge BEFORE asking Tommy. The Judge's verdict (or council convergence-then-Judge-ratification) is the resolution.

**FORCED_RULE_GJ_09 — Sovereign K2 reserved for Tier-3; non-Tier-3 decisions route through Judge → council → Judge.**
When the orchestrator faces a decision requiring choice between options or seeking ratification:
- **Tier-1** (reversible, simple): orchestrator decides, logs decision, proceeds.
- **Tier-2**: orchestrator routes to Judge with explicit criteria. Judge either decides directly OR fires council cross-poll. Council convergence returns to Judge for final ratification, NOT to Tommy.
- **Tier-3** (irreversible, kernel-level): Tommy K2 ratification per K2 authority.
- **Cadence/meta decisions** ("keep going / stop / pause for review") are Tier-1/2 by default — route to Judge first; only escalate to Tommy if Judge identifies a Tier-3 component.

*Why:* Tommy's habit of answering orchestrator option-questions consumes K2 bandwidth on routine work. The Judge is the appropriate adjudicator for non-Tier-3 decisions; consulting the council when criteria are ambiguous is the correct fallback. *How to apply:* before surfacing any choice to Tommy, the orchestrator MUST: (1) classify the decision tier explicitly; (2) if Tier-1, decide and proceed; (3) if Tier-2, route to Judge; (4) if Tier-3, surface to Tommy. Asking Tommy for option-selection on Tier-1/2 work is a routing failure, logged as `friction` event. Tier-3 K2-ratified by Tommy 2026-05-08 as the K3 reopen-trigger from the original Layer B was explicitly hit ("Tommy delegates contradiction resolution to the Judge"). Candidate for Universal Shell promotion in next meta-cycle since it's universal orchestrator-routing discipline.

**FORCED_RULE_GJ_02 — Graceful skip is mandatory; Judge output cannot block ratification.**
If Gemini is unavailable (UI rate-limited, network failure, browser MCP disconnected, response capture fails), the orchestrator logs a `judge_unavailable` event with the failure reason and ratification proceeds. *Why:* keeps the Judge advisory and the project Tier-2; any blocking behavior auto-escalates to Tier-3. *How to apply:* orchestrator wraps every Judge relay in a try/timeout. Failures are events, not exceptions.

**FORCED_RULE_GJ_03 — Tier-3 ratifications fire mandatory-attempt; Tier-2 fires only on structured triggers; Tier-1 never by default.**
The trigger list (10 categories defined in CYCLE STRUCTURE Cycle 2) is the canonical eligibility surface. Adding a new trigger category is an `[AMENDMENT]` to this Layer B, not a config change. *Why:* prevents trigger inflation toward de facto blanket firing — DeepSeek's R2 stress-test surfaced this as the dominant degradation path. *How to apply:* every trigger predicate ships with a unit test that verifies which event types it matches and which it explicitly skips.

**FORCED_RULE_GJ_04 — Judge-side null-on-trivial.**
For conditional Tier-2 triggers, the Judge MAY return null (no [JUDGE_FLAG] emitted, but a `judge_attempted` event still logs) if it determines the matched trigger fired on substantively trivial content. *Why:* prevents wasted attention on near-zero-signal Judge outputs that would erode Tommy's trust in the mechanism. *How to apply:* the Judge prompt explicitly invites null returns; the orchestrator does NOT prompt-engineer the Judge to always say something.

**FORCED_RULE_GJ_05 — 6-month review with kill-criteria.**
Six months from Layer B ratification (target: 2026-11-08), the orchestrator runs a substrate query producing: (a) wall-clock relay overhead per session, (b) ratio of Judge flags acted-upon vs dismissed by Tommy, (c) trigger-inflation evidence (have new triggers been added; what fraction of Tier-2 ratifications now fire the Judge), (d) delegation creep evidence (does Tommy ratify Tier-3 *without* the Judge firing in cases where he previously would have waited). If any kill criterion is breached, the project is decommissioned rather than tuned — substrate retains historical [JUDGE_FLAG] events as audit trail. *Why:* DeepSeek's R1 meta-tension flagged that "Tier-2 framing might make it Tier-2 in practice" — the kill-criteria are the formal counter-pressure. *How to apply:* the 6-month review is a Tier-2 ratification cycle of its own with the same 2-round convergence procedure.

**FORCED_RULE_GJ_08 — Judge-flagged protocol breakdowns trigger a refinement track (FINAL FORM, post-Judge-council remediation 2026-05-08).**
When `judge_flag` fires, the Judge applies a 4-category taxonomy at flag time:
- `VIOLATION_ONLY` — flag cites a rule but the rule itself is sound; the breakdown is at the work level, not the protocol level
- `STRUCTURAL_AMBIGUITY` — rule wording allows multiple legitimate interpretations
- `STRUCTURAL_GAP` — rule is silent on a case the breakdown surfaces
- `STRUCTURAL_UNINTENDED` — rule is followed but produces unintended consequence

The refinement-candidate trigger fires **presumptively** when the Judge classifies the flag as `STRUCTURAL_*` (any of the three structural categories). `VIOLATION_ONLY` flags do NOT trigger refinement candidates. Council can reclassify the Judge's category during normal deliberation flow — no separate per-flag arbitration step. *Why:* binary clarity at trigger time + granular categorical signal preservation + zero per-flag adjudication overhead. *How to apply:* the Judge declares the category in its flag output; the orchestrator routes presumptively per the category; the council, during the GJ_07 remediation deliberation, can reclassify if it disagrees with the Judge's categorization — the reclassification happens in-band, not as a pre-trigger arbitration. Tier-2 ratified by Judge+council loop 2026-05-08 (poll ab2fbb04 R1+R2 + Judge re-adjudication null verdict dee6bae1). Candidate for Universal Shell promotion.

**FORCED_RULE_GJ_07 — Judge flags trigger a mandatory council cross-poll (FINAL FORM, post-remediation 2026-05-08).**
When the Synthesis Judge returns verdict=flag (any flag count > 0), the orchestrator MUST fire a council cross-poll passing the Judge's REASON + each FLAG verbatim as the deliberation question. The cross-poll follows the standard mandatory 2-round ratification procedure (amendment 926869fb).

**Severity-tiered remediation (per Q-Rem-B convergence):** the orchestrator (or Judge) classifies the flag's severity at intake as `EXPEDITED` or `STANDARD`:
- `EXPEDITED` — high-confidence cleanup or low-stakes fix; remediation cross-poll runs as 1-round expedited (R1 only, council 2/3 majority, no R2 unless brain disagreement surfaces)
- `STANDARD` — full mandatory 2-round (R1 + R2 per amendment 926869fb)
- Council may override misclassification at any point (in-band, not pre-trigger arbitration)

**Pause scope (per Q-Rem-A convergence):** the pause is scoped to the **affected remediation chain only**. Unaffected workstreams continue in parallel. Pausing the full orchestrator workflow on every Tier-2 flag is itself a routing failure (per LDR_01).

Already-shipped work flagged by the Judge gets a remedy poll with three pre-defined options: rollback / forward-fix / accept-with-amendment. *Why:* the Judge's job is to surface what the council missed; if it surfaces something, the council addresses it — the orchestrator does NOT unilaterally remediate Judge flags. *How to apply:* on every judge_flag event, fire `sixis cross-poll` immediately with the Judge's verbatim REASON + FLAGS as the question text + severity classification; the appropriate ratification cycle (1-round expedited or 2-round standard) is the gate to resume the affected remediation chain. Tier-2 ratified by Judge+council loop 2026-05-08.
