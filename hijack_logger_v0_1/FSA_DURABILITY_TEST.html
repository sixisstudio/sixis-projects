Phase 0 Gate 5 — File System Access API Durability Test

This page tests whether Chrome's File System Access API can sustain 500 small appends to a file and survive a Chrome restart with one-click re-grant. Per the council R2 ratification, this gate decides Q2 (FSA vs native messaging) empirically.

Step 1 — Pick a directory and write 500 appends

Click the button below. Pick any directory (Desktop is fine — we'll clean up after). The page will create fsa_test.txt there and append 500 lines to it, one at a time, with a small await between each (mimicking the per-hand write pattern).

Step 2 — Close and reopen Chrome

After Step 1 reports DONE, quit Chrome entirely (Cmd+Q on Mac, Alt+F4 on Windows — not just close the tab). Reopen Chrome, navigate back to this page (it lives at ~/Documents/Claude/Projects/SixiS/projects/hijack_logger_v0_1/FSA_DURABILITY_TEST.html — Cmd+O to open file, or drag the file into a Chrome window).

Step 3 — Re-grant the handle and verify
